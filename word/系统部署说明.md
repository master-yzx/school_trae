# 校园二手交易平台 - 部署说明文档

> 本文档说明项目所需技术栈、运行环境，以及在新机器上从零部署本系统的完整步骤（含 MySQL、Redis、Maven、JDK 等配置）。

---

## 1 所需技术栈与软件版本

1. **操作系统**
   - 支持：Windows 10/11 或 Linux（如 Ubuntu / CentOS），推荐 64 位系统。

2. **JDK**
   - 版本要求：**JDK 17**（与 `campusServer/pom.xml` 中 `<java.version>17</java.version>` 一致）
   - 验证命令：
     ```bash
     java -version
     ```

3. **Maven**
   - 版本：3.6+（推荐 3.8.x）
   - 用途：构建后端 Spring Boot 项目 `campusServer`
   - 验证命令：
     ```bash
     mvn -version
     ```

4. **Node.js 与 npm**
   - Node.js：16+ 或 18+
   - npm：随 Node 一起安装
   - 用途：构建与运行前端项目 `campusWeb`
   - 验证命令：
     ```bash
     node -v
     npm -v
     ```

5. **MySQL 数据库**
   - 版本：MySQL 8.x（兼容 5.7，要求支持 `utf8mb4` 编码）
   - 默认数据库名：`campus_market`（在初始化脚本中自动创建）

6. **Redis**
   - 版本：Redis 5+ / 6+ 均可
   - 用途：
     - 存储登录图形验证码
     - 存储刷新 token 相关数据（如有）

7. **其他依赖（通过 Maven / npm 自动下载）**
   - 后端：
     - Spring Boot 3.x（web、data-jpa、websocket、aop、validation 等）
     - Spring Data Redis
     - MySQL 驱动 `mysql-connector-j`
     - JWT 库 `jjwt`
   - 前端：
     - Vue 3
     - Element Plus
     - Pinia
     - Axios
     - ECharts

## 2 数据库与 Redis 初始化

### 2.1 MySQL 初始化

1. **创建数据库**

   在 MySQL 控制台执行：

   ```sql
   CREATE DATABASE IF NOT EXISTS campus_market
     DEFAULT CHARACTER SET utf8mb4
     COLLATE utf8mb4_general_ci;
   ```

2. **执行初始化脚本**

   在项目根目录下，找到 `db/db_init.sql`，在命令行执行：

   ```bash
   mysql -u root -p campus_market < db/db_init.sql
   ```

   或使用图形管理工具（如 Navicat、DataGrip）打开 `db_init.sql` 并执行。

3. **确认关键数据表存在**

   主要表包括（但不限于）：

   - 用户与角色：
     - `user_account`
   - 商品与媒体：
     - `product`
     - `product_media`
   - 购物车与收藏：
     - `cart_item`
     - `favorite_category`
     - `favorite_item`
     - `browse_history`
   - 订单与日志：
     - `order_main`
     - `order_item`
     - `order_log`
   - 聊天与论坛：
     - `chat_session`
     - `chat_message`
     - `forum_post`
     - `forum_comment`
   - 公告与系统配置：
     - `notice`
     - `system_config`
   - 站内消息与操作日志：
     - `message`
     - `operation_log`

### 2.2 Redis 启动与配置

1. 安装并启动 Redis（以本机为例）：

   ```bash
   # Linux 示例
   redis-server
   ```

2. 默认连接信息：
   - 主机：`127.0.0.1`
   - 端口：`6379`
   - 无密码（如设置了密码，需要在应用配置中同步配置）

3. 检查 Redis 是否可用：

   ```bash
   redis-cli ping
   # 返回 PONG 表示正常
   ```

## 3 后端部署（campusServer）

### 3.1 配置文件（application.yml）

在 `campusServer/src/main/resources/` 下，找到或创建 `application.yml` 或 `application-prod.yml`，示例配置如下：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/campus_market?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8
    username: your_mysql_user
    password: your_mysql_password
    driver-class-name: com.mysql.cj.jdbc.Driver

  jpa:
    hibernate:
      ddl-auto: none    # 使用 db_init.sql 初始化，不自动建表
    show-sql: true
    properties:
      hibernate.format_sql: true

  data:
    redis:
      host: 127.0.0.1
      port: 6379
      # password: your_redis_password

server:
  port: 8080
```

> 说明：如使用多环境配置（dev/prod），可将上述配置放入对应 profile 文件，并通过 `--spring.profiles.active=prod` 指定。

### 3.2 使用 Maven 构建

在 `campusServer` 目录下执行：

```bash
mvn clean package -DskipTests
```

打包完成后，在 `campusServer/target/` 目录下会生成类似：

```text
campusServer-0.0.1-SNAPSHOT.jar
```

### 3.3 启动后端服务

方式一：使用 Maven 插件（开发环境）：

```bash
mvn spring-boot:run
```

方式二：运行打包后的 Jar（推荐部署时使用）：

```bash
java -jar target/campusServer-0.0.1-SNAPSHOT.jar
```

如果使用特定 profile：

```bash
java -jar target/campusServer-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

### 3.4 验证后端接口

启动后，可在浏览器或 Postman 中访问：

- `http://localhost:8080/api/system/config/footer`  
  - 若返回站点底部配置 JSON，说明后端正常。
- `http://localhost:8080/api/products/...`  
  - 可以进一步验证商品接口是否可用。

## 4 前端部署（campusWeb）

### 4.1 安装依赖

进入 `campusWeb` 目录：

```bash
cd campusWeb
npm install
```

### 4.2 开发模式运行

```bash
npm run dev
```

默认 Vite 会在 `http://localhost:5173`（或提示的端口）启动前端开发服务器。

此时通过前端的 `baseURL: '/api'` 代理请求后端 `http://localhost:8080/api/...`。

### 4.3 生产构建

```bash
npm run build
```

构建产物会生成在 `campusWeb/dist/` 目录，包含 HTML、JS、CSS 等静态文件。

## 5 前后端分离部署（结合 Nginx 的推荐方案）

### 5.1 后端常驻进程

在服务器上，推荐使用 `nohup` 或 `systemd`：

```bash
nohup java -jar campusServer-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod > server.log 2>&1 &
```

### 5.2 Nginx 配置示例

假设：
- 前端构建产物路径：`/var/www/campus/dist`
- 项目根目录下 `upload/` 用于存放：
  - 商品图片/Logo
  - 导出的 CSV（订单、日志、统计报表）

示例 `nginx.conf` 片段：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/campus/dist;
    index index.html;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 反向代理后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 映射上传文件（图片、导出文件等）
    location /upload/ {
        alias /path/to/project/upload/;
        autoindex off;
    }
}
```

> `/path/to/project/upload/` 替换为后端运行所在项目根目录下的 `upload` 绝对路径。

## 6 Upload（上传目录）说明

系统中上传文件（商品图片、Logo、导出的 CSV 等）由工具类 `UploadPaths` 统一管理：

- 代码位置：`campusServer/src/main/java/com/campus/common/utils/UploadPaths.java`
- 逻辑：
  - 若当前工作目录为 `campusServer` 或 `campusWeb`，则回退到上一级作为项目根目录；
  - 在根目录下创建/使用 `upload/` 目录；
  - 所有上传/导出文件都写入该目录。

部署时需确保：

- 后端进程对 `upload/` 目录拥有读写权限；
- Nginx 或其他 Web 服务器正确映射 `/upload/` URL 到此目录。

## 7 常见问题与排查

1. **前端能打开页面，但接口报 404 或 502**
   - 检查后端是否启动成功（端口 8080 是否在监听）。
   - 检查 Nginx `proxy_pass` 地址是否指向正确端口。

2. **登录时报“登录已过期，请重新登录”**
   - 确认浏览器中是否携带 `Authorization: Bearer ...` 头（前端 axios 自动处理）。
   - 若长期不操作，token 过期，重新登录即可。

3. **Redis 连接失败**
   - 查看后端日志是否有 `RedisConnectionException`。
   - 检查 Redis 服务是否已启动、地址/端口/密码是否正确。

4. **Excel 打开导出 CSV 乱码**
   - 系统已对 CSV 写入 UTF-8 BOM，一般不会乱码；
   - 若仍显示为 `####`，可在 Excel 中调宽列宽或选择“数据 → 自文本导入”指定 UTF-8。
