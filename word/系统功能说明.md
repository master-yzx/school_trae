# 校园二手交易平台 - 功能说明文档

> 本文档说明当前系统已实现的主要功能，并给出每个功能在前后端对应的核心文件位置，便于查阅与维护。

## 1 系统概述

本系统为校园二手交易平台，包含前台用户端与后台管理端两部分，支持二手商品的发布、浏览、收藏、购物车结算、订单管理、在线咨询（聊天）、论坛交流、公告展示，以及后台对商品、用户、订单、公告、统计、日志等进行统一管理。

- 前端项目：`campusWeb`
- 后端项目：`campusServer`
- 数据库脚本：`db/db_init.sql`
- 文件上传： `upload`

## 2 前台功能

### 2.1 用户认证模块

**功能点**

- 用户注册（手机号 + 密码 + 角色）
- 图形验证码登录
- 忘记密码 / 重置密码
- 退出登录

**前端文件**

- 登录/注册页面：`campusWeb/src/views/auth/AuthIndex.vue`
- 忘记密码页面：`campusWeb/src/views/auth/ForgotPassword.vue`
- 头部登录入口与用户菜单：`campusWeb/src/components/layout/AppHeader.vue`
- 认证状态 Store：`campusWeb/src/stores/auth.js`
- HTTP 请求封装（axios + token 拦截器）：`campusWeb/src/utils/request.js`

**后端文件**

- 认证控制器：`campusServer/src/main/java/com/campus/modules/auth/controller/AuthController.java`
- 认证服务实现：`.../modules/auth/service/impl/AuthServiceImpl.java`
- 认证 DTO：`.../modules/auth/dto/*.java`
- JWT 处理：`.../modules/auth/jwt/JwtService.java`
- 刷新 token 存储：`.../modules/auth/jwt/RefreshTokenStore.java`
- 当前登录用户上下文：
  - `com.campus.common.auth.JwtAuthFilter`
  - `com.campus.common.auth.CurrentUserHolder`

**主要技术栈**

- 前端：Vue 3、Element Plus、Pinia、Axios
- 后端：Spring Boot 3、Spring MVC、Spring Validation、Spring Data JPA、Redis（图形验证码）、JWT（登录态）
- 数据库：MySQL（`user_account`、`system_config` 等表）

### 2.3 购物车与订单

#### 2.3.1 购物车管理

**功能点**

- 商品详情页加入购物车
- 购物车列表展示：标题、价格、数量、卖家、校园等
- 数量增减、删除条目、清空购物车

**前端文件**

- “加入购物车”逻辑：`ProductDetail.vue` 中 `handleAddCart`
- 购物车列表页：`campusWeb/src/views/cart/CartList.vue`

**后端文件**

- 控制器：`campusServer/src/main/java/com/campus/modules/cart/controller/CartController.java`
- Service：`.../modules/cart/service/impl/CartServiceImpl.java`
- 实体/DTO：`CartItem.java`、`CartItemDTO.java`
- 数据表：`cart_item`（`db/db_init.sql` 中第 10 部分）

**主要技术栈**

- 前端：Vue 3 + Element Plus（`el-table`、`el-input-number`）、Pinia/本地状态管理
- 后端：Spring Boot 3 + Spring MVC + Spring Data JPA（购物车项增删改查）
- 数据库：MySQL（`cart_item` 表，与 `user_account`、`product` 关联）

#### 2.3.2 下单与购物车合并结算

**功能点**

- 商品详情页单件下单（立即购买）
- 购物车中勾选多个“同一卖家”的商品，一次性合并为一个订单：
  - 订单主表记录总金额、总数量、买卖双方
  - 订单明细表按商品拆分记录

**前端文件**

- 单件下单：`ProductDetail.vue` 中 `handleBuy`
- 购物车结算：`CartList.vue` 中 `settle`
- 用户订单列表：`campusWeb/src/views/order/OrderList.vue`
- 订单详情页：`campusWeb/src/views/order/OrderDetail.vue`
  - 展示订单基本信息
  - 展示 `items` 中每个商品明细、买家/卖家信息

**后端文件**

- 用户侧订单接口：`campusServer/src/main/java/com/campus/modules/order/controller/OrderController.java`
  - `POST /api/orders/create`：单商品下单
  - `POST /api/orders/create-from-cart`：购物车合并下单
  - `GET /api/orders/user` / `seller`：订单列表
  - `GET /api/orders/{id}`：订单详情
  - 多个 `/orders/{id}/...` 操作接口：取消、确认收货、售后、发货、支付
- 订单业务实现：`.../modules/order/service/impl/OrderServiceImpl.java`
  - `createOrder`、`createOrderFromCart`、`getDetail`、`pageUserOrders` 等
  - 使用 `ResponseStatusException` 返回明确错误（未登录、商品不存在、不能购买自己商品等）
- DTO：
  - `OrderDetailDTO.java`（含 `List<OrderItemDTO> items`）
  - `OrderItemDTO.java`
  - `OrderListItemDTO.java`
  - `OrderLogDTO.java`
- 实体与表：
  - `OrderMain.java` ↔ `order_main`
  - `OrderItem.java` ↔ `order_item`
  - `OrderLog.java` ↔ `order_log`

**主要技术栈**

- 前端：Vue 3、Vue Router（订单详情路由）、Element Plus（表格/对话框/分页）、Axios
- 后端：Spring Boot 3、Spring MVC、Spring Data JPA（多表关联查询）、统一异常处理（`ResponseStatusException`）
- 数据库：MySQL（`order_main`、`order_item`、`order_log`）

### 3.7 操作日志与导出

**功能点**

- 自动记录后台 `/api/admin/**` 非 GET 请求的操作日志（通过 AOP）
- 支持分页查询、查看详情
- 导出日志为 CSV（Excel 直接打开不乱码）

**前端文件**

- 管理页：`campusWeb/src/admin/log/AdminOperationLog.vue`

**后端文件**

- 实体/表：`OperationLog.java` ↔ `operation_log`
- Repository：`OperationLogRepository.java`
- Service：
  - `AdminLogService.java`、`AdminLogServiceImpl.java`（分页、详情、导出）
  - `OperationLogRecorder.java`（封装写日志逻辑）
- 控制器：`AdminLogController.java`
- AOP 切面：`AdminOperationLogAspect.java`
  - 拦截 `com.campus.modules.admin..controller..*` 中的非 GET 请求
  - 根据 URL 归类 type（系统配置/订单操作/商品审核等），并记录 IP、结果、时间等。
